02-1 훈련 세트와 테스트 세트

무언가 판단이 필요한 데이터가 주어진다면 머신러닝 알고리즘을 학습시킬 수 있고 이를 훈련이라고 합니다. 이때 사용되는 데이터는 입력과 타깃 그리고 특성으로 표현할 수 있습니다. 입력은 판단을 위한 기초데이터 타깃은 판단의 결과 특성은 판단의 기준으로 삼을 수 있는 규칙성입니다. 입력과 타깃 데이터가 있으며 머신러닝 알고리즘이 정답을 가리킬 수 있는 지도 학습supervised learning을 하면 됩니다. 그런데 만약 입력 데이터만 있고 타깃 데이터가 없다면 비지도 학습unsupervised learning을 하면 됩니다.

그래서 훈련 세트와 테스트 세트는 왜 분류한거냐고요? 만약 확보한 모든 데이터를 훈련에 사용하면 훈련시킨 머신러닝 알고리즘을 제대로 검증할 수 없기 때문입니다. 그래서 반드시 훈련 세트와 테스트 세트를 분리하여 훈련된 결과물을 검증하는데 사용해야합니다. 꼭 기억합시다.

훈련 세트와 테스트 세트!


샘플링 편향과 넘파이Numpy

훈련 세트와 테스트 세트를 만들때 샘플들이 골고루 섞여 있지 않아서 전체 데이터를 데표할수 없는 현상을 샘플링 편향sampling bias이라고합니다. 확보된 데이터가 특성feature에 따라 샘플들이 골고루 섞여 있지 않다면 직접 섞어줍니다. 파이썬 라이브러리인 넘파이Numpy를 사용하면 손쉽게 임의로 섞인 데이터를 만들 수 있습니다. 만든 데이터를 넘파이 배열로 바꿔준 후 shuffle함수를 사용하면 됩니다. 하지만 이렇게 해도 골고루 잘 섞인다는 보장은 사실 없습니다.
